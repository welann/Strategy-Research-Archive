## 主题

这篇文章试图把普通用户对收益的理解，从“资产会不会涨”升级为：

**我承担了什么风险，市场为什么愿意为这种风险付钱，这笔收益经过费用、对冲和尾部损失后还剩多少。**

作者用四组内容建立这个框架：

1. 现货与永续对冲，收取资金费率或基差；
2. 提供 AMM 流动性，再动态对冲变化中的 Delta；
3. 用 Pendle `PT / YT` 拆分本金和未来收益；
4. 用金库封装策略、净值、权限、赎回和风险披露。

文章最值得保留的判断是：

**Delta 中性只消除了或降低了一阶方向暴露，不会消除资金费率、Gamma、流动性、清算、交易所、智能合约、预言机、管理人和赎回风险。**

但这篇文章同时承担 City Protocol 的产品叙事功能。通用知识与项目方主张需要分开：

- Delta、资金费率、负 Gamma、PT/YT 和金库尽调框架可以独立使用；
- `TaaS / VaaS / NaaS` 是否真正提供文中描述的能力，需要产品、代码、审计和运行数据验证；
- 文中没有提供 City Protocol 金库的真实净值、回撤、赎回记录或策略表现。

## 核心框架

### 1. 不要把收益等同于价格上涨

大多数加密账户天然具有正 Delta：

- 现货做多；
- 山寨币高 beta 做多；
- 杠杆多头；
- 以高相关抵押品借款后继续做多。

这些策略在牛市里看起来不同，实际上常常共享同一个收益来源：市场上涨。

当市场下跌时，它们会一起失效。

作者建议把收益拆成其他可能来源：

- 资金费率；
- 期现基差；
- 交易手续费；
- 期限折价；
- 借贷利差；
- 流动性补偿；
- 波动率风险溢价；
- 市场微结构错误。

这种思路不是追求“完全没有风险”，而是把方向风险替换成更明确、可计量的其他风险。

### 2. Delta中性是局部属性，不是产品标签

Delta 衡量组合价值对标的价格小幅变化的一阶敏感度。

例如：

```text
多 1 ETH 现货：Delta 约 +1
空 1 ETH 永续：Delta 约 -1
组合净 Delta：约 0
```

但“当前净 Delta 为零”不代表未来一直为零。

Delta 可能因为以下因素变化：

- LP 资产比例变化；
- 期权 Gamma；
- 现货与衍生品标记价格不同；
- 保证金资产本身涨跌；
- 资金流入流出；
- 对冲腿部分成交；
- 借贷或赎回改变名义敞口。

所以 Delta 中性是一项持续维护的状态，不是开仓后永久成立的标签。

## 资金费率与基差策略

### 1. 基础结构

最简单的结构是：

```text
买入现货
       +
做空等名义价值永续
       =
降低方向暴露
```

如果永续资金费率为正：

- 多头向空头付费；
- 永续空头获得资金费率；
- 现货腿用于对冲价格上涨风险。

这笔收入可以理解为市场对杠杆多头需求的定价。

### 2. 示例年化建立在费率不变的假设上

文章举例：

```text
0.01% / 8 小时 × 3 × 365 = 10.95%
```

这个计算在数学上正确，但它假设：

- 每次结算费率长期维持 `0.01%`；
- 没有费率反转；
- 仓位一直完整；
- 没有资金闲置；
- 没有交易成本、借贷成本和税费；
- 没有复利或资金调度差异。

因此 `10.95%` 是简单情景年化，不是可直接承诺的收益。

更合理的净收益表达是：

```text
净收益
= 实际收到的资金费率
- 开平仓手续费
- 滑点
- 借贷成本
- 资金闲置成本
- 对冲误差
- 运营与托管损失
```

### 3. 回本持有期比名义年化更实用

文章给出一个很好的执行例子：

- 总进出摩擦：`280 美元`
- 每日资金费率收入：`30 美元`

回本持有期：

```text
280 / 30 ≈ 9.3 天
```

这意味着策略必须在第十天之后仍能维持足够收益，才真正开始赚钱。

所以入场前不能只看当前资金费率，还要估计：

- 费率持续时间；
- 费率分布和反转概率；
- 基差变化；
- 退出深度；
- 持仓期间所需保证金。

### 4. 组合保证金提高的是ROE，也放大系统脆弱性

作者指出，机构可能通过组合保证金、跨仓和抵押复用，把基础 `10%` 收益转成更高的资本回报率。

这里需要区分：

- `Return on notional`：相对策略名义规模的收益；
- `Return on equity`：相对实际占用自有资本的收益。

如果用更少保证金承载同样名义仓位，ROE 会提高，但策略本身的毛收益没有凭空增加。

同时，资本缓冲变小会放大：

- 标记价格偏差；
- 保证金规则变化；
- 交易所临时提高风险参数；
- 对冲腿无法平仓；
- 极端行情追加保证金；
- 抵押品与仓位相关性跳变。

高 ROE 本质上是高资本利用率，不是免费收益。

## 对冲型LP

### 1. LP手续费是承担库存和波动风险的补偿

AMM LP 收取手续费，但自动做市机制会改变资产比例：

- 价格上涨时卖出上涨资产；
- 价格下跌时买入下跌资产；
- 相对单纯持有产生无常损失。

文章引用恒定乘积 AMM 的无常损失公式：

```text
IL = 2 × sqrt(r) / (1 + r) - 1
```

其中 `r` 是价格比。

价格翻倍或减半时，标准全区间恒定乘积仓位的 IL 约为 `-5.72%`。

但这个公式不能直接代表 Uniswap V3 集中流动性仓位。

集中流动性还依赖：

- 区间上下界；
- 初始价格位置；
- 是否离开区间；
- 再平衡路径；
- 手续费复投；
- 多次调整产生的实现损益。

### 2. LP的Delta会随价格变化

如果 LP 当前等效持有 `20 ETH`，可以做空约 `20 ETH` 永续降低方向暴露。

但价格变化后：

- ETH 上涨，LP 中 ETH 数量减少；
- ETH 下跌，LP 中 ETH 数量增加；
- 原有空头不再匹配新的 Delta。

这就是为什么对冲型 LP 需要动态再平衡。

### 3. 负Gamma意味着对冲可能买高卖低

LP 仓位通常具有负 Gamma 特征：

- 上涨后，原空头可能过多，需要买回；
- 下跌后，LP 的 ETH 增加，需要继续加空。

动态对冲可能形成：

```text
上涨时买回
下跌时加空
```

也就是局部买高卖低。

LP 手续费只有覆盖以下成本后，才形成净收益：

- 无常损失；
- 动态对冲损耗；
- gas；
- 滑点；
- MEV；
- 脱离区间后的闲置；
- 极端行情库存风险。

### 4. 再平衡频率是核心控制变量

频率过低：

- Delta 漂移；
- 方向暴露增大；
- 极端行情可能来不及修复。

频率过高：

- 手续费、gas 和滑点增加；
- 负 Gamma 成本更频繁实现；
- 订单更容易遭遇逆向选择。

所以专业系统需要按波动率、流动性和剩余区间动态设置再平衡阈值。

## Pendle PT与YT

### 1. PT把未来本金折价变成定期收益

文章把 `PT` 类比为零息债券：

- 当前以折价买入；
- 到期兑换底层资产；
- 折价收敛形成收益。

简化年化公式：

```text
隐含简单年化
= (1 / PT价格 - 1) × (365 / 剩余天数)
```

例如：

```text
PT价格 = 0.95
剩余期限 = 180 天
简单年化约 10.67%
```

需要注意：

- 这是简单年化，不是有效复利年化；
- 底层生息资产未必稳定锚定 `1`；
- 到期兑换依赖底层协议和 Pendle 合约；
- 提前退出价格受流动性和收益率曲线影响。

### 2. YT是对未来收益率的杠杆敞口

`YT` 代表到期前的收益权。

如果未来实际收益高于市场隐含收益，YT 可能获利；反之可能快速损失价值。

作者用 `PT + YT = 底层资产` 解释经济关系。这适合作为入门直觉，但实际交易还会受到：

- 标准化收益资产结构；
- AMM 定价；
- 收益累积；
- 到期时间；
- 激励和积分；
- 流动性；
- 底层资产价格；

影响，不能简单把 `YT` 当成固定倍数收益杠杆。

### 3. PT的“固定收益”仍带信用与技术风险

PT 锁定的是相对底层资产的到期兑换关系，不代表法币本金绝对安全。

需要评估：

- 底层资产脱锚；
- 收益来源协议被攻击；
- 智能合约漏洞；
- 到期赎回故障；
- 预言机；
- 二级流动性；
- 提前退出折价。

## 组合与风险度量

### 1. 组合重点是收益来源之间的相关性

作者建议不要只问哪个策略 APY 最高，而要问：

- 多个策略是否在同一极端行情一起失效；
- 是否依赖同一交易所、稳定币或预言机；
- 赎回周期是否同时变长；
- 尾部损失是否来自同一触发器。

名义上不同的策略可能共享风险因子：

- 现货 + 永续和对冲型 LP 都可能依赖同一 CEX；
- 多个 PT 可能依赖同一稳定币；
- 多个金库可能共享同一管理人或预言机；
- 不同协议可能共享同一跨链桥。

真正的分散需要拆到底层依赖。

### 2. Sharpe和Sortino都无法完整描述跳跃风险

`Sharpe` 使用总波动，`Sortino` 更关注下行波动。

但加密收益策略常见：

- 平时小赚；
- 净值平滑；
- 偶尔因脱锚、攻击或冻结出现大额跳跃损失。

历史标准差可能严重低估这种尾部。

还需要观察：

- 最大回撤；
- 最差单日 / 单周损失；
- Expected Shortfall；
- 压力情景损失；
- 流动性调整后回撤；
- 赎回暂停时间；
- 坏账和损失恢复率。

### 3. Kelly依赖可靠的概率与收益分布

分数 Kelly 比满 Kelly 保守，但它仍要求：

- 期望收益估计可信；
- 方差和相关性稳定；
- 尾部风险没有被遗漏；
- 策略容量足够；
- 交易成本可预测。

对样本少、负偏度、状态切换明显的策略，即使 `25%-50% Kelly` 也可能过大。

所以 Kelly 应作为仓位上限参考，而不是自动下注公式。

## 金库基础设施

### 1. 金库真正封装的是运营复杂度

用户看到的可能只是：

- 存入；
- APY；
- 风险标签；
- 赎回。

系统实际需要处理：

- 策略权限；
- 账户和托管；
- 净值计算；
- 预言机与价格验证；
- 申购赎回；
- 费用；
- 策略白名单；
- 异常暂停；
- 对账与审计记录；
- 容量限制。

文章的核心观点是：普通用户不需要手动执行复杂策略，但必须有足够信息判断自己购买了什么风险。

### 2. 可审计不等于无须信任

如果策略使用：

- 中心化交易所；
- 链下托管；
- 场外借贷；
- 延迟更新的策略估值；
- 管理人提交的外部数据；

链上金库只能展示部分状态。

需要额外验证：

- 交易所账户资产；
- 未实现盈亏；
- 负债和保证金；
- 管理人是否能转移资金；
- NAV 数据是否完整；
- 链下头寸是否有第三方证明。

把金库份额代币化，不会自动解决底层资产真实性和偿付能力。

### 3. 赎回机制是收益产品的核心风险变量

高收益往往伴随：

- 资金锁定；
- 策略平仓时间；
- 较差的退出深度；
- 固定结算周期；
- 提前赎回费用；
- 极端行情暂停。

需要确认：

- 是即时赎回还是周期赎回；
- 赎回使用哪个时点 NAV；
- 排队和容量如何处理；
- 谁能暂停；
- 暂停多久；
- 恢复顺序；
- 策略亏损由谁承担。

### 4. City Protocol部分属于产品方叙事

作者将 City Protocol 定位为：

- `TaaS`：代币化即服务；
- `VaaS`：金库服务；
- `NaaS`：新金融服务。

文章描述了理想功能：

- 策略代币化；
- 净值和证明；
- 权限与白名单；
- 申购赎回；
- 钱包、交易终端和金融科技应用集成。

这些是产品目标，不是文章已经证明的交付结果。

后续验证应看：

- 合约代码和部署；
- 审计；
- 实际金库；
- NAV 更新记录；
- 管理权限；
- 历史赎回；
- 异常事件处理；
- 策略真实回撤。

## 高收益产品七问

文章给出的七问是全文最适合直接复用的部分。

### 1. 收益来源是什么

区分：

- 资金费率；
- 手续费；
- 借贷利差；
- PT 折价；
- RWA 利息；
- 代币补贴；
- 方向收益。

补贴 APY 和结构性现金流不应使用相同估值。

### 2. 方向风险是否被对冲

如果声称 Delta 中性，应说明：

- 对冲工具；
- 对冲频率；
- 允许的 Delta 区间；
- 对冲失败条件；
- 极端行情降杠杆规则。

### 3. 主要尾部风险是什么

要求产品明确最可能伤害本金的事件：

- 脱锚；
- 攻击；
- 清算；
- 交易所冻结；
- 预言机；
- 流动性枯竭；
- 管理人错误；
- 跨链桥事故。

### 4. NAV如何计算

确认：

- 数据来源；
- 更新频率；
- 估值模型；
- 多方验证；
- 异常偏离处理；
- 未实现盈亏和负债是否计入。

### 5. 赎回机制是什么

确认：

- 频率；
- 通知期；
- 结算时间；
- 赎回价格；
- 排队；
- 暂停权限；
- 极端情况下的退出顺序。

### 6. 策略边界是什么

确认管理人是否受到：

- 市场白名单；
- 资产白名单；
- 仓位上限；
- 杠杆上限；
- 回撤限制；
- 函数权限；
- 多签和时间锁；

约束。

### 7. 产品是否可验证

可验证材料可能包括：

- 链上头寸；
- 储备或账户证明；
- NAV 历史；
- 结算记录；
- 审计；
- 第三方确认；
- 事故和暂停历史。

“链上”本身不是答案，关键是核心资产、负债和权限是否都能被验证。

## 风险与限制

### 1. 文章没有提供任何策略实盘绩效

文章解释了机制，但没有提供：

- 实际净收益；
- 最大回撤；
- 资金费率分布；
- LP 对冲记录；
- 金库 NAV；
- 压力测试；
- 赎回数据。

所以它是一份教育和产品定位文章，不是策略业绩证明。

### 2. “熊市双位数收益”容易掩盖状态依赖

熊市中：

- 资金费率可能转负；
- 成交量可能下降；
- LP 手续费可能不足；
- PT 底层资产可能脱锚；
- 借贷利率和清算风险可能上升；
- 流动性最差时赎回最困难。

双位数收益只在特定市场状态和风险承担下成立。

### 3. 复杂性被封装后，用户更容易忽略风险

金库让操作变简单，但可能让用户只看到 APY。

如果产品没有主动披露：

- 底层仓位；
- 对冲状态；
- 杠杆；
- 对手方；
- 赎回期限；
- 权限；

封装反而会增加信息不对称。

### 4. 多策略组合可能产生隐藏的风险集中

多个“中性”策略可能同时依赖：

- USDC；
- 同一家交易所；
- 同一托管人；
- 同一预言机；
- 同一 Curator；
- 同一跨链桥。

组合标签多样不等于风险来源多样。

## 扩散分析 / 延展思路

### 1. 用收益分解表替代单一APY

可以要求每个产品披露：

```text
净收益
= 资金费率 / 基差
+ 手续费
+ 期限折价
+ 借贷利差
+ 代币激励
+ 方向 PnL
- 对冲成本
- 交易成本
- 管理费
- 尾部损失
```

这样才能判断收益主要来自稳定现金流、补贴还是未披露方向风险。

### 2. 为每个策略建立压力场景

至少模拟：

- BTC / ETH 单日上涨或下跌 `20%-40%`；
- 资金费率连续转负；
- 稳定币脱锚；
- CEX 提现暂停；
- LP 离开区间；
- gas 飙升；
- 预言机延迟；
- 金库集中赎回；
- PT 底层协议暂停。

压力测试比历史 Sharpe 更接近真实尾部风险。

### 3. 建立可验证性分层

可以把金库分为：

- 全部头寸链上可见；
- 链上资产 + 链下对冲；
- 链下托管、链上份额；
- 仅由管理人报告 NAV。

层级越往下，越需要第三方证明、审计和赎回约束。

### 4. 把赎回负债纳入策略风险

金库不仅有资产端策略，还有负债端赎回承诺。

如果用户可每日赎回，而底层策略需要一周平仓，就存在期限错配。

应跟踪：

- 流动资产比例；
- 正常和压力情景平仓时间；
- 单日赎回容量；
- 排队规模；
- 备用流动性；
- 暂停触发条件。

## 一句话结论

**Delta 中性不是无风险收益，而是把方向风险换成资金费率、Gamma、流动性、对手方和运营风险；真正可靠的收益产品必须同时说明现金流来源、对冲机制、净值、权限、尾部风险和赎回路径。**
