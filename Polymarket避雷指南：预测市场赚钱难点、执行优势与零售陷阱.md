## 主题

这篇内容在讲：**普通人进入 Polymarket 这类预测市场，真正面对的不是“判断事件对不对”的简单 50/50 游戏，而是一个利润高度集中、执行质量极端重要、费用和延迟会持续吞掉边际优势的专业化市场。**

作者真正想表达的是：

- Polymarket 不是适合普通人稳定赚生活费的地方；
- 零售亏钱的主要原因不是只因为“预测错”，而是买错价格、进场太晚、抄单延迟、忽略费用、和 HFT 抢速度；
- 预测市场里的真正优势通常来自执行、定价模型、限价单等待、品类偏差和资金成本，而不是单纯“研究更多”；
- AI 可以辅助筛选和过滤坏交易，但不能直接全自动替你赚钱；
- 如果仍然要做，只能在少数低拥挤、低资金效率、低速度要求的边角策略里找机会。

需要注意：原文中“3 条可行路径”的具体方法和“文献溯源表”都被作者隐藏了，所以这篇更像风险框架和避坑清单，不是完整可复制的交易手册。

## 作者的判断方法

### 1. 先看赚钱概率：稳定盈利者极少

作者把 Polymarket 的第一层现实定义为：绝大多数人亏钱。

他给出的核心数字是：

- 约 `2.5 million` 用户中；
- 只有 `172` 人做到了“稳定 passive income”；
- 对应比例约 `0.007%`。

这组数字的作用不是精确预测你是否会赚钱，而是打破一个常见误解：

**预测市场不是“我对事件有观点，所以我就有 50% 胜率”的地方。**

它更接近一个高度专业化的交易场所，少数人依靠执行和结构性优势拿走大部分利润。

### 2. 再看零售最容易踩的第一类坑：买便宜 longshot YES

作者把 `1-10c YES longshot` 列为零售亏钱的第一大来源。

他的逻辑是：

- 便宜 YES 看起来赔率高；
- 零售用户容易被“万一发生就几十倍”的叙事吸引；
- 但低价 YES 的真实发生概率往往低于价格隐含概率；
- 这相当于持续缴纳“乐观税”。

作者引用 Kalshi 数据说明，`1c YES` 合约实际胜率只有 `0.43%`，并强调 Polymarket 不同品类偏差并不完全一样，但买便宜 YES longshot 仍然是零售最容易亏钱的行为之一。

关键点在于：

**低价格不等于高赔率；如果真实概率比价格更低，便宜合约反而是最贵的合约。**

### 3. 再看执行结构：预测准不等于赚钱

作者引用 Della Vedova 的结论，给出全文最重要的市场机制判断：

> 预测准确率高于随机的人可能亏钱，因为他们来得太晚；接近随机准确率的人反而可能通过执行质量赚钱。

这句话是全文的核心。

作者想说明，预测市场的收益不只来自“最终事件对不对”，还来自：

- 你以什么价格进场；
- 你是不是 maker；
- 你是否能等到 dip fill；
- 你是否比别人更早反应；
- 你的仓位大小和市场深度是否匹配；
- 你是否被手续费吃掉。

所以普通人常犯的错误是：

**把预测市场理解成观点市场，而不是订单簿市场。**

### 4. 再看赚钱者画像：不是高胜率，而是盈亏结构好

作者分析了同时出现在 `30d + 7d + 1d` 排行榜 top50 的 `14` 个钱包。

这些钱包的特征包括：

- `68%` 做 Sports；
- `86%` 在 `0.30-0.70` 的中间价位交易；
- `50%` maker-heavy，用限价单等待成交；
- 单市场胜率只有 `41%-51%`。

这说明赚钱者未必是“每场都猜对”的人。

他们更像是：

- 亏的时候亏小；
- 赢的时候赢大；
- 成交价格更好；
- 交易品类和订单方式更有优势。

这和传统交易里的“低胜率高盈亏比”逻辑很接近。

### 5. 最后看品类偏差：不同市场不能套同一套规则

作者特别强调 Weather 和 Political 的定价偏差方向不同：

- Weather 市场短期偏 overconfident，favorite 可能被高估；
- Political 市场可能存在 underconfidence；
- Sports 校准很好，市场标 `70%` 的比赛大约真的有 `70%` 胜率。

这说明 Polymarket 不是一个统一市场，而是一组不同微结构的品类市场。

同一种策略在不同品类上的方向可能完全相反。例如：

- 在某些政治市场，买 favorite 可能有结构性优势；
- 在天气市场买 favorite 可能系统性亏钱；
- 在体育市场，除非你有执行优势或真实模型优势，否则没有免费午餐。

### 一句话总结判断方法

作者的判断链条是：**先承认利润极度集中，再拆解零售亏钱的行为模式，最后把预测市场理解为“执行质量 + 品类偏差 + 费用结构 + 延迟优势”的交易场所，而不是单纯判断事件对错的投票游戏。**

## 作者的应对策略

### 策略 1：先排除 7 类明确不该做的事

作者给出的主要应对策略首先是负面清单。

不要做：

- 买 `1-10c YES longshot`；
- 做 generic LP / market making；
- 让 AI 全自动交易；
- 抄 top wallet 的单；
- 做普通跨平台套利；
- 做 speed / latency trading；
- 在 Weather 市场上机械买 favorite。

这份清单的价值在于，它不是告诉你哪里赚钱，而是先告诉你普通人最可能在哪里亏钱。

### 策略 2：不要做 generic LP，除非你有品类定价 alpha

作者认为，单纯“挂单收 reward”的 generic maker 策略在 2026 年已经失效。

原因包括：

- 作者自己的样本显示 aggregate maker gross return 为负；
- poly-maker 作者也承认当前环境不再盈利；
- LP rewards 设计稿并不等于实盘 P&L；
- LLM 真金白银比赛全亏。

唯一可能例外是：

**你有具体品类的定价模型，例如体育模型，然后用 maker 方式拿更好的成交价格。**

也就是说，maker 只是执行方式，不是 alpha 本身。

### 策略 3：AI 只能辅助，不应自动下单

作者引用 Prediction Arena 的实验，说明多个 frontier LLM 真金白银交易仍然全部亏损。

他的结论是：

- AI 可以用来帮助选市场；
- AI 可以帮助过滤明显坏交易；
- AI 可以帮助做研究和整理信息；
- 但不要把下单权完全交给 AI。

更直接地说：

**AI 可以提升研究效率，但不能自动替代定价能力、风控能力和执行能力。**

### 策略 4：不要抄单，因为你抄不到“时间”

作者承认，Polymarket 的 data API 可以看到 top wallet 的交易，也能追踪持续盈利钱包。

但他认为抄单不可行，原因是：

- 你至少慢 `30` 秒；
- 他们很多成交来自限价单等待 dip fill；
- 你看到时，最佳价格已经不存在；
- 他们的单笔规模和你的资金规模不匹配；
- 他们单市场胜率本来也不高，你抽样抄几笔会承受很大噪声。

这句话可以压缩成：

**Top wallet 的 edge 不是“买了什么”，而是“什么时候、什么价格、用什么方式成交”。**

### 策略 5：如果仍然要做，只能找 HFT 不屑做的边角

作者虽然整体偏劝退，但也列出三条“可行路径”：

- `97-99c Bonding`
- `Political Favorite`
- `Multi-outcome Arbitrage`

不过具体方法被隐藏了。

从正文透露的信息看，作者认为可行机会大致有几个共同特征：

- 资金锁定时间长；
- 单位回报很低；
- HFT 资金成本不划算；
- 需要利用品类校准偏差；
- 不靠速度抢新闻。

例如，他提到 `97-99c bonding` 的本质可能是锁 `97` 美元数周赚 `1` 美元，这类机会收益低到 HFT 不感兴趣，但对低资金成本、小额实验者可能有意义。

## 风险与限制

### 1. 原文可行路径隐藏，不能直接复制

这篇文章有很强的避坑价值，但不是完整策略说明。

作者把三条可行路径的具体方法隐藏了，所以读者不能只凭标题就执行：

- 不知道筛选条件；
- 不知道入场规则；
- 不知道止损和资金管理；
- 不知道样本外表现；
- 不知道策略容量。

### 2. 文中很多外部研究结论需要独立核验

作者列出大量论文、API、链上样本和源码审计结果，但附录文献表被隐藏。

因此在归纳时只能把它们当作“作者引用的研究依据”，不能直接当作已经完成独立复核的事实。

### 3. 费用结构会改变策略生死线

作者强调 `2026-03-30 fee expansion` 后，不同品类 taker fee 明显不同：

- Sports：`3%`
- Crypto：`7.2%`
- Politics / Finance：`4%`
- Weather / Culture：`5%`
- Geopolitics：`0%`

同时手续费公式为：

```text
fee = rate × price × (1 - price)
```

这意味着中间价位费用最高，极端价格费用更低。很多纸面小 edge 策略，如果没有把 fee 放进去，结果会完全反转。

### 4. 盈利截图和高收益故事有强幸存者偏差

作者提醒，不要相信 X 上“有人赚了多少”的帖子。

原因是：

- 很多人只展示 gross，不扣亏损；
- 只展示 peak，不展示后续回撤；
- 2024 年有效的方法，2026 年可能已经失效；
- 赛道拥挤后，原有收益会被竞争吃掉。

### 5. 预测市场的集中风险容易被低估

作者用 LLM 曾经盈利后单日大亏的例子提醒：多个相关 weather positions 同时结算，会让表面分散的仓位变成同一方向暴露。

这类风险在预测市场里很常见：

- 多个事件共享同一宏观驱动；
- 多个天气市场共享同一气候模型误差；
- 多个政治市场共享同一新闻冲击；
- 多个体育市场共享同一信息源或赔率偏差。

## 扩散分析 / 延展思路

### 1. 把 Polymarket 当训练场，而不是收入来源

这篇最重要的实用结论是降低目标。

更合理的定位是：

- 用小额资金学习订单簿；
- 验证概率校准；
- 练习限价单和费用意识；
- 观察不同品类的市场微结构。

不要把它当作稳定现金流工具。

### 2. 对普通人更现实的第一步：只做记录，不先交易

如果想进入 Polymarket，可以先做无风险观察：

- 选一个品类，例如政治或体育；
- 每天记录市场价格、自己估计概率、最终结果；
- 比较自己的 Brier score 与市场价格；
- 只在连续证明自己有定价优势后再小额交易。

这比直接入场买合约更接近“先证明 edge，再承担风险”。

### 3. 如果做 maker，必须先有定价模型

不要把 maker 当成收益来源。

正确顺序应该是：

```text
先有品类模型
再知道合理概率
再用 limit order 改善成交
最后才考虑 maker / reward
```

如果没有前两步，挂单只是给更懂价格的人提供流动性。

### 4. AI 更适合做过滤器，而不是交易员

结合作者观点，一个更实用的 AI 用法是：

- 自动汇总市场说明和结算规则；
- 标记定义模糊的市场；
- 找出相互相关的仓位；
- 提醒费用、流动性和滑点；
- 帮你生成交易后复盘。

这比让 AI 直接下注更稳健。

### 5. 真正值得研究的是“品类偏差”，不是全站通吃策略

作者多次强调不同品类校准不同。

所以更有价值的研究方向不是：

“有没有一个 Polymarket 万能策略？”

而是：

- Sports 是否真的完全校准；
- Weather favorite 是否持续被高估；
- Political favorite 的 underconfidence 是否仍然存在；
- Geopolitics 免手续费是否带来不同微结构；
- Multi-outcome 市场是否还有定义偏差或价格闭合机会。

## 一句话结论

这篇文章的核心是：**Polymarket 对普通人最危险的地方不是预测难，而是执行慢、费用高、价格差、样本噪声大和赛道拥挤；在没有品类定价能力和执行优势之前，最好的策略往往不是进场赚钱，而是先避开会系统性亏钱的坑。**
