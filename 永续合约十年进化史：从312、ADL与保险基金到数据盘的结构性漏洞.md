## 主题

这篇内容在讲：**今天看起来已经很“常识化”的永续合约规则，并不是交易所一开始就设计得很完美，而是每一次大事故之后被迫补出来的风险工程。**

作者真正想表达的是：

- 标记价格、资金费率、阶梯式清算、保险基金、`ADL`、`U` 本位、组合保证金，这些都不是优雅设计，而是灾难留下的伤疤；
- 永续合约市场已经从极客赌场进化成全球定价机器，但它并没有变成一个完全中性的基础设施；
- 主流资产的核心矛盾已经从“方向性赌场”变成“资本效率与风险控制”；
- 长尾资产的核心矛盾则从合约公式本身，转移到了更上游的现货流动性、指数输入质量和代币发行结构；
- 最新一轮最难解决的问题，不是再调一遍资金费率，而是**高 `FDV`、低流通、强控盘代币把永续合约的整个精密引擎变成了操纵放大器。**

所以这篇文章本质上不是“永续合约科普”，而是一篇**用十年灾难史串起永续合约机制史的市场结构文章。**

## 作者的判断方法

### 1. 先用“灾难驱动的机制演进”解释规则为何存在

作者不是按产品手册来讲永续合约，而是按事故时间线来讲：

- 为什么需要指数价格；
- 为什么要有标记价格；
- 为什么社会化回拨会被淘汰；
- 为什么保险基金成了标准配置；
- 为什么 `U` 本位最终替代币本位；
- 为什么资金费率机制会从固定结算，走向动态升频，再走到底层公式重构。

这套方法的好处是：**规则不再是黑箱参数，而是有清晰历史来源的风险补丁。**

### 2. 用“定价三件套”解释永续合约最初如何站住脚

作者把永续合约创世阶段压缩成三件套：

- 指数价格：现货锚；
- 标记价格：清算锚；
- 资金费率：收敛锚。

其中最关键的理解不是公式本身，而是三者各自解决一个不同问题：

- 指数价格负责对抗单点污染；
- 标记价格负责减少插针猎杀；
- 资金费率负责让没有交割日的合约仍能向现货收敛。

### 3. 把币本位反向合约视为早期永续的“原罪”

作者对早期币本位合约的判断非常明确：

- 牛市时，它通过合约盈利和抵押品升值叠加，放大了收益；
- 熊市时，它也会让合约亏损和保证金贬值同频共振，形成自反性的死亡螺旋。

也就是说，作者不是把币本位看成中性产品选择，而是看成一种**在极端下跌中天然会加速爆仓链条的结构。**

### 4. 用 OKEx 50000 BTC 事件解释仓位上限、保险基金与 ADL 的必要性

作者把 `2018-08-03` 的 `OKEx` 大仓位事故当成第一堂大课。

他从这件事里抽出的不是单一结论，而是三件风控基础设施的必要性：

- 仓位上限：防止单一账户变成“大而不能倒”；
- 保险基金：先吸收穿仓亏损，而不是把损失摊给无辜盈利用户；
- `ADL`：如果保险基金不够，至少让损失定向落到高杠杆高浮盈对手盘，而不是全市场大锅饭。

这里作者还特意强调了一个历史细节：**`ADL` 的原始发明者不是 `BitMEX`，而是更早期的 `Huobi`。**

### 5. 用 312 解释为什么永续合约从“合约设计”转向“风险工程”

`2020-03-12` 是全文最关键的拐点。

作者把 `312` 暴露的问题拆成三层：

- 币本位合约的自反性崩塌；
- 清算级联和做市商撤退导致的流动性真空；
- 公链拥堵让理论上的套利修复无法落地。

这三层叠加后，行业不得不接受两个变化：

- 主流产品从币本位反向合约转向 `USDT / USDC` 线性合约；
- 风险工程从幕后支持环节，变成了衍生品产品设计的核心学科。

### 6. 用机构入场解释主流永续为何从赌场变成套利机

到了 `2020-2023` 阶段，作者开始用市场参与者结构来解释机制变化。

他的核心观察是：

- 机构和做市商大规模进入后，主流品种的资金费率被持续压平；
- 盘口价差被显著压缩；
- `OI` 规模跃升；
- 组合保证金和统一账户让资本效率显著提升。

作者特别把 `Bybit` 的弯道超车归因到这一点：

- 先推 `Portfolio Margin`；
- 再推 `Unified Trading Account`；
- 用资本效率而不是纯手续费竞争来抢机构心智。

这说明作者判断交易所竞争格局时，看的是**保证金效率、机构承载能力和用户结构**，不只是市场份额。

### 7. 用 TRB 和 ALPACA 说明“资金费率与规则本身也会被武器化”

这是全文最有启发性的部分之一。

作者认为，在高控盘、薄流动性的标的上，资金费率不一定还是“价格收敛工具”，它可能变成：

- 对拥挤空头的持续抽血工具；
- 对庄家现货拉盘的收益放大器；
- 在下架、升频、流动性撤退这些条件叠加后，被规则本身放大的攻击向量。

`TRB` 说明：负资金费率可以被控盘者拿来反向收租。  
`ALPACA` 说明：下架公告和动态升频结合后，连“交易所的风控规则”本身都可以被反向利用。

### 8. 把“妖币”与“数据盘”看成问题从合约端转移到输入端

到了 `MYX / COAI / AIA` 和后续“数据盘”阶段，作者的判断明显升级了：

- 过去的问题主要在合约端规则如何设计；
- 现在的问题更上游，变成现货流动性、指数来源、流通盘结构和筹码集中度。

作者认为，当一个代币本身具备：

- 高 `FDV`
- 低流通
- 高持仓集中度

那么合约引擎再精密，也只是帮这种操纵结构做放大，而不是制衡。

### 9. 最后用交易所格局和 OI/成交量比解释不同平台的“用户心智”

文章后段并没有停在产品机制，还继续上升到平台层面：

- `Binance / OKX` 更偏高换手短线；
- `Bybit / Gate` 有更高比例的中期套利与机构持仓；
- `KuCoin` 的高 `OI / 交易量` 比更像低换手高沉淀结构；
- `Bitget` 不是靠风险引擎创新，而是靠跟单把复杂产品翻译给散户。

也就是说，作者看交易所不是只看排名，而是看**用户类型、仓位周期和平台叙事方式。**

### 一句话总结判断方法

作者的判断链条是：**先用灾难史解释永续合约每条规则的来历，再用参与者结构解释主流资产为何机构化，随后用 `TRB / ALPACA / 数据盘` 证明风险已经从合约公式本身转移到流动性、指数和发行结构，最后再用交易所用户结构解释不同平台的博弈风格。**

## 作者的应对策略

这篇文章不是直接给你一套交易系统，但它隐含了非常清楚的应对框架。

### 策略 1：不要把交易所规则当成黑箱参数

作者最重要的隐含建议是：

- 理解每一个参数为什么存在；
- 看到资金费率、标记价格、`ADL`、保险基金、下架过渡规则时，不要只把它们当公告细节；
- 它们本身就是市场结构的一部分，也是潜在风险源的一部分。

### 策略 2：主流资产更适合做资本效率和基差结构，不适合继续当纯方向赌场

在 `BTC / ETH` 这类主流资产上，作者的框架明显更偏向：

- 基差套利；
- 资金费率套利；
- 组合保证金下的高资本效率对冲；
- 用更低摩擦成本运行系统化仓位。

换句话说，主流永续已经不再是最适合零售方向性硬赌的桌子。

### 策略 3：长尾资产先看现货流动性、指数构成和流通盘，再决定要不要参与

对于长尾标的，作者的核心建议其实偏保守：

- 先看现货是不是薄；
- 再看指数是不是脆；
- 再看流通盘是不是小；
- 再看筹码是不是集中；
- 最后才看方向。

如果这几个条件同时脆弱，那么永续盘口的价格本身就可能不是“市场观点”，而是操纵放大后的结果。

### 策略 4：把下架、费率公式调整、结算频率变化当成事件，而不是噪音

`TRB` 和 `ALPACA` 的共同点是：

- 规则不是背景；
- 规则本身就是事件变量。

这意味着实战上不能把以下内容当噪音忽略：

- 资金费率结算频率变化；
- 单次费率上限变化；
- 标记价格算法调整；
- 下架过渡安排；
- 杠杆上限和持仓上限收紧。

### 策略 5：回避高 FDV、低流通、强控盘且已上永续的代币

作者虽然没有直接写成一句“不要碰”，但他的逻辑已经很明确：

- 这类标的不是普通高波动标的；
- 它们是结构性脆弱、容易被控盘并跨所污染指数的放大器。

如果一定要参与，也应该默认：

- 仓位更小；
- 期限更短；
- 对冲更快；
- 对流动性和规则变化更敏感。

### 2. 关于“按持仓时间加权费率”的评论，提出了一个值得继续想的方向

评论区有读者提出：

- 应该优先惩罚新近拥挤的一侧；
- 而不是让更早建仓、方向上反而更可能提供风险承接的仓位一起被无差别抽血。

这个思路未必容易工程化，但它确实切中了一个难点：**现有资金费率机制更擅长惩罚“偏离结果”，不一定擅长区分“偏离是谁新制造出来的”。**

### 3. “永续合约是交易所最赚钱也最危险的基础设施”这句评论很贴题

另一条评论把永续合约概括为：

- 对交易所最赚钱；
- 对小币永续最容易演化成“双轨制”放大器。

这虽然不是严格的方法论补充，但非常准确地补出了全文气质：**永续合约不是天然中性基础设施，它既能提高市场效率，也能在错误场景里放大失真。**

## 风险与限制

### 1. 这是一篇高质量历史综合，但不是可直接下单的量化策略

作者提供的是：

- 机制解释；
- 结构观察；
- 风险工程逻辑；
- 典型事件模板。

它没有提供完整的：

- 入场规则；
- 仓位公式；
- 止损规则；
- 交易成本模型。

### 2. 结构判断正确，不代表长尾标的上能稳定赚钱

即使你看对了：

- 某个币的现货极薄；
- 某个指数容易被污染；
- 某个空头拥挤会被轧；

你仍然可能因为：

- 流动性太差；
- 交易所规则突变；
- 庄家控盘时间超预期；
- 借贷和资金费率成本过高

而在执行层面亏损。

### 3. 文中很多数字更适合作为结构证据，不适合作为机械阈值

比如：

- 保险基金规模；
- `OI / 交易量` 比；
- 长尾标的 `OI` 与现货量级差；
- 资金费率极值频率。

这些指标用来理解结构非常好，但很难单独抽出来做机械化开仓阈值。

### 4. 文章是 CEX 中心视角，对链上永续的差异着墨较少

作者最后提到了一种可能路径：

- 把做空机制迁移到链上；
- 让做空直接作用于真实流动性池，而不是中心化指数。

但全文主体仍然是 `CEX perp` 视角，所以对链上永续如何重新定义这些问题，只点到为止。

## 扩散分析 / 延展思路

### 1. 更保守的用法：把这篇文章当成长尾永续避坑框架

如果你不做复杂套利，也不碰组合保证金，这篇依然有很强实用价值。

最直接的落地方式是：

- 看到长尾永续先不看故事；
- 先看现货流动性；
- 再看流通盘；
- 再看持仓集中度；
- 再看指数来源是否脆弱。

只要这几项足够糟糕，就把它当作潜在的结构性操纵盘，而不是普通高波动交易机会。

### 2. 更进一层的研究方向，不该只盯资金费率，而应做“上币质量评分”

如果沿着作者逻辑往前推，未来真正有价值的研究，不一定是再回测一次资金费率，而是做一套：

- 流通率评分；
- 现货交易所分散度评分；
- 指数输入质量评分；
- 前二十地址集中度评分；
- `OI / 现货成交量` 脆弱度评分；
- 下架与风控规则评分。

也就是把“代币发行结构风险”前置成一个系统筛选器。

### 3. 对主流资产，机会可能更多来自 carry 与对冲，而不是继续押纯方向

文章里一个很重要但容易被忽略的结论是：

- 机构把主流永续的无脑偏离压得越来越薄；
- 主流桌子上最稳定的机会，可能更多来自利差、费率、库存对冲和资本效率；
- 真正容易出现超大偏离的地方，往往又是最不适合普通人硬上的长尾桌子。

这意味着：

- 主流资产更像低边际 alpha、高执行门槛；
- 长尾资产更像高表面波动、高操纵风险。

### 4. 一个值得继续比较的方向：链上借贷做空能否比中心化指数空头更稳

作者最后提到，数据盘问题可能逼着一部分做空机制迁移到链上。

这背后的可研究问题是：

- 如果空头不是建立在中心化指数上，而是建立在真实池子流动性和借贷库存上；
- 那么控盘者的攻击路径、成本结构和上限会不会变化；
- 这种变化到底会让市场更公平，还是只是把战场从 `CEX` 搬到了链上。

这是后续很值得单独展开的一条线。

## 一句话结论

**现代永续合约不是一套从一开始就优雅成立的中性基础设施，而是一台在一次次爆仓、穿仓、操纵与规则反制中长出来的风险工程机器，而它当下最棘手、最未解决的漏洞，已经从合约公式本身转移到了代币发行结构和指数输入质量。**
