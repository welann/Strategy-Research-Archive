## 主题

这篇文章在讲一个 DeFi 借贷协议设计问题：链上借贷已经把“利率怎么定”做得越来越复杂，但对“抵押品下跌时怎么避免硬清算悬崖”改进不够。

作者提出把传统金融里的 `CPPI`（Constant Proportion Portfolio Insurance，固定比例投资组合保险）迁移到链上借贷，让抵押品不再是静态的一篮子风险资产，而是一个会根据安全垫自动调仓的“风险资产 + 稳定币”组合。

它的目标不是消灭风险，而是把普通借贷里的单点清算线，改造成一个随价格下跌逐步减风险的过程。借款人仍然可能亏损、牺牲上行、承受跳空风险，但不会因为一根普通价格针直接进入全额清算。

## 问题定义

### 1. 链上借贷主要创新在利率，清算机制仍偏粗糙

作者先把链上借贷创新分成两条轴：

- 第一条轴是“信用成本怎么定价”：从 `Compound / Aave` 的变量利率，到 `Pendle` 的固定收益拆分，再到 `Morpho Midnight` 这种固定利率、固定到期结构。
- 第二条轴是“抵押品下跌时怎么处理”：大部分协议仍然依赖硬清算，即价格跌破某条线后，清算人以折扣关闭仓位。

作者认为第一条轴已经有大量创新，第二条轴反而相对空白。

普通硬清算的问题是：

- 抵押品下跌到阈值后，仓位被一次性关闭。
- 借款人付出清算罚金。
- 如果资产一周后反弹，借款人已经失去参与反弹的仓位。
- 协议虽然保护了贷款人，但把风险集中在一个价格点上。

所以这篇文章真正想解决的问题不是“贷款利率更低”，而是“抵押品下跌路径能不能更平滑”。

### 2. Curve 和 f(x) 已经证明软清算方向可行

作者认为 `Curve crvUSD` 的 `LLAMMA` 和 `f(x) Protocol` 的 `Liquidation Brake` 都已经在做类似的事。

`LLAMMA` 的核心不是一条清算线，而是把清算区域拆成多个价格 band：

- 价格下跌穿过某个 band 时，部分抵押品被逐步换成 crvUSD。
- 价格回升穿过 band 时，crvUSD 又重新买回抵押品。
- 套利者通过 oracle 偏移报价完成再平衡。

`f(x)` 的 `Liquidation Brake` 则是在杠杆仓位接近清算价时，燃烧部分债务、卖出部分抵押资产，把杠杆率拉回安全区域，而不是立刻关闭整个方向敞口。

作者把这两类机制统一解释为“抵押品组合保险”：下跌时去风险，回升时重新增加风险敞口。

## 实现思路 / 系统设计

### 1. CPPI 的核心公式

CPPI 的核心变量很少：

```text
cushion = portfolio value - floor
risky exposure = m * cushion
```

其中：

- `floor` 是组合不能跌破的底线。
- `cushion` 是当前组合价值高于底线的安全垫。
- `m` 是 multiplier，决定承担多少风险资产敞口。
- 风险资产可以是 `ETH / BTC wrapper / SOL` 等抵押品。
- 安全资产通常是稳定币。

当 cushion 很厚时，系统可以持有更多风险资产；当 cushion 被下跌吃掉时，系统机械地把风险资产换成稳定币；当 cushion 恢复时，系统再逐步买回风险资产。

把这个逻辑放进借贷协议里，就是让抵押品自己防御清算。

### 2. floor 不应等于 debt，而应高于 debt

作者强调 floor 应该设在债务上方，而不是刚好等于债务。

原因是链上再平衡不是瞬时完成的：

- swap 有滑点。
- keeper 或套利者有执行延迟。
- oracle 可能滞后或被操纵。
- 价格可能跳空。
- MEV 可能放大再平衡成本。

如果 floor 就等于 debt，一旦再平衡没赶上，抵押品可能直接跌穿债务，协议立即产生坏账。

因此更合理的设计是：

```text
floor = debt + gap margin
```

当抵押品价值接近 floor 时，CPPI 篮子应该已经基本转成稳定币。floor 到 debt 之间的空间，则作为硬清算 backstop 和清算人激励来源。

### 3. multiplier 是借款人的风险旋钮

作者给的简化例子是：

- 抵押品价值：`100`
- 借款：`70`
- 暂不考虑 gap margin
- floor：`70`
- cushion：`30`

如果借款人希望初始 100% 暴露在 BTC 上，则：

```text
100 = m * 30
m = 3.33
```

这意味着 multiplier 不是装饰参数，而是整个产品的风险形状。

它同时决定两件事：

- 初始风险资产敞口有多少。
- 能承受多大的价格跳空风险。

作者把跳空容忍度近似写成：

```text
gap tolerance = 1 / m
```

所以：

- `m = 2`：初始风险资产敞口更低，稳定币比例更高，能承受约 50% 的跳空，但上行参与更少。
- `m = 3.33`：可以接近满仓风险资产，能承受约 30% 的渐进式下跌窗口。
- `m = 5`：上行参与更强，但跳空容忍度约 20%，更容易在剧烈行情中失效。

这个设计的产品化价值在于：借款人不再只能接受协议统一清算参数，而是可以明确选择自己想买多少下行保护、愿意放弃多少上行。

### 4. 协议实现需要四个核心模块

作者设想的协议更接近一个 isolated lending market。

核心模块包括：

- 抵押品篮子：不是纯风险资产，而是风险资产 + 稳定币的 CPPI managed basket。
- 再平衡引擎：可以类似 `LLAMMA` 用报价吸引套利者，也可以用 keeper、batch auction 或 `CoW` 风格 settlement 减少 MEV 和滑点。
- floor 与 hard liquidation backstop：floor 设在 debt 上方，floor 和 debt 之间用于清算激励和债权保护。
- 坏账准备金：从借款人 spread 中抽取，用来覆盖极端 gap 风险。

利率本身可以是变量利率，也可以是固定利率。作者偏好的完整形态是固定利率 + 固定期限 + 自我防御抵押品，因为这更接近传统资本保护产品。

## 关键技术点 / 难点

### 1. 这个机制不是免费保护

CPPI 最大的误读是把它当作“更安全且不损失收益”的借贷。

实际上它只是改变风险形状。

成本主要有三类：

1. whipsaw bleed：价格在区间里来回震荡时，系统会下跌卖出、上涨买回，等于反复低卖高买。
2. forgone upside：如果系统在低位转成稳定币，后续反弹时借款人不会完整参与。
3. gap risk：价格跳空太快时，系统来不及再平衡，仍然可能产生坏账。

作者引用 `crvUSD` 的软清算经验来说明这种成本是真实存在的，不是理论问题。

### 2. CPPI 天然有“下跌卖出”的反馈风险

传统金融里，1987 年黑色星期一经常被拿来讨论 portfolio insurance 的反馈效应。

严格说，当年被指责的主要是 synthetic put / dynamic hedging 类型的策略，不完全等同于 CPPI。但它们共享一个危险特征：下跌时需要卖出风险资产。

如果大量仓位使用类似机制，市场下跌时再平衡卖盘本身可能放大下跌。

这对链上设计有直接含义：

- multiplier 不能只按用户偏好设定，还要按资产流动性设上限。
- 再平衡频率不能无限提高，因为越频繁越可能增加 MEV 和交易成本。
- 资产准入不能太宽，否则长尾抵押品会在跳空时击穿 floor。
- 协议需要限制同质化 CPPI 仓位在同一资产上的集中度。

### 3. Oracle、MEV 和流动性是核心工程问题

CPPI 公式本身很简单，难点在链上执行。

一个可用协议必须回答：

- 用什么 oracle 判断 cushion？
- oracle 价格和可成交价格偏离时如何处理？
- 谁来执行再平衡？
- keeper 不执行或作恶怎么办？
- 再平衡是否会被 MEV 夹击？
- 稳定币篮子用什么资产？
- 风险资产流动性不足时如何限制新仓？
- 同一价格区间内大量仓位同时卖出时如何排队？

所以这篇文章真正的工程难点，不是把公式写进合约，而是把“公式产生的动态交易需求”安全地落到链上市场里。

### 4. hard liquidation 仍然不能取消

作者没有把 CPPI 包装成无清算借贷。

相反，他明确认为必须保留 hard liquidation backstop。

原因很直接：

- CPPI 能处理渐进式下跌。
- CPPI 不能处理无法成交的跳空。
- 不能把 lender 的安全完全押在动态再平衡上。
- gap margin 和 reserve 只能降低坏账概率，不能消灭坏账。

因此，合理产品不是“没有清算”，而是“多数普通回调由软再平衡吸收，极端断层仍由硬清算和准备金兜底”。

## 关键补充

### 1. 作者定位：这是给借贷协议建设者看的提案

作者在回复中明确说，如果你在做 lending protocol，这篇值得花十分钟读。

这说明它不是面向普通用户的收益策略，而是一个协议 primitive 提案。读它时应该重点关注机制设计、参数化和失败模式，不应该直接理解成“某种更安全的借贷产品已经可用”。

### 2. Options 是另一条实现路线

`Panoptic_xyz` 的评论指出，类似目标也可以通过 options 实现：保留下行保护，同时保留上行参与。

这条评论很重要，因为 CPPI 和 options 的风险形状不同：

- CPPI 通过动态交易复制保护，成本体现在震荡损耗、滑点和 gap risk。
- options 通过预先购买凸性获得保护，成本体现在期权权利金和流动性。

如果链上期权市场足够深，option overlay 可能比频繁再平衡更干净；如果期权市场不深，CPPI 这种现货/稳定币篮子反而更容易落地。

### 3. Bitmor 类似方案提示：预买期权可替代部分再平衡成本

`megabyte0x` 的评论提到，`Bitmor's Loan protocol` 曾有类似 `f(x) + fixed duration` 的设计：贷款开始时买入 option，用期权保护替代价格波动中频繁买稳定资产。

这个补充提供了一个很有用的对照：

- 作者方案更像动态保险。
- Bitmor 评论里的方案更像静态保险。
- 动态保险省掉显性期权费，但付出交易路径成本。
- 静态保险提前锁定保护，但依赖期权定价、期限、流动性和履约结构。

这也是协议设计时最值得比较的分叉点。

## 风险与限制

### 1. 不要把“自我防御”理解成“不会亏”

CPPI loan 的目标是防止抵押品在普通下跌中直接撞上硬清算线。

它并不保证：

- 抵押品价值不下降。
- 借款人不亏钱。
- 借款人完整参与反弹。
- 协议不会产生坏账。
- 极端行情里一定能完成再平衡。

更准确的说法是：它把清算悬崖换成连续成本，把一次性罚没换成一系列可观察、可定价、但仍然昂贵的防御动作。

### 2. 最怕高波动横盘

单边缓慢下跌时，CPPI 有机会逐步减风险。

但高波动横盘可能是最差场景之一：

- 下跌时卖出风险资产。
- 回升时重新买入风险资产。
- 价格反复穿越再平衡区间。
- 用户既没有真正避开大跌，也不断支付交易损耗。

这类 whipsaw bleed 会让用户觉得“没有被清算，但资产慢慢漏掉了”。

因此，产品界面必须把预期 bleed 和历史模拟说清楚，否则用户很容易误以为它是免费保险。

### 3. 长尾资产不适合一开始就接入

作者也倾向于从 blue chip collateral 和保守 multiplier 开始。

这是合理的，因为 CPPI 对资产连续可交易性要求很高。

不适合早期接入的资产包括：

- 深度不足的小盘币。
- oracle 容易被操纵的资产。
- 流动性高度集中在少数池子的资产。
- 经常跳空或停摆的封装资产。
- 流动性和市值错配严重的高 FDV 资产。

这类资产看起来很需要“自我防御借贷”，但恰恰最容易在 gap risk 里击穿机制。

### 4. 协议可能制造新的系统性卖压

如果一个资产上大量借款都采用类似 CPPI 规则，那么价格下跌时，它们会同时降低风险资产敞口。

这可能形成链上版本的 portfolio insurance feedback loop：

```text
价格下跌
-> cushion 下降
-> CPPI 篮子卖出风险资产
-> 现货卖压增加
-> 价格继续下跌
-> 更多篮子触发再平衡
```

所以协议不能只做单仓风险控制，还要做系统级风险控制：

- 限制单资产 CPPI 借款总量。
- 对 multiplier 设全局上限。
- 在高波动时提高 gap margin。
- 极端行情下暂停新开仓或降低新仓 multiplier。
- 对再平衡排队、批量成交和拍卖机制做设计。

## 扩散分析 / 延展思路

### 1. 这类协议真正卖的是“风险形状”

传统 DeFi 借贷卖的是流动性：

- 用户抵押资产。
- 借出稳定币。
- 接受清算线。

CPPI 借贷卖的不是更高杠杆，而是更平滑的风险形状：

- 用户牺牲一部分上行。
- 接受动态调仓损耗。
- 换取不被普通下跌直接硬清算。

这和传统结构化产品的逻辑更接近：不是追求最高预期收益，而是让风险更符合保守资金能接受的形状。

### 2. 可以形成三种产品路线

围绕“抵押品自我防御”，大概有三条路线：

1. CPPI 动态篮子：用风险资产和稳定币再平衡来管理 floor。
2. Option overlay：贷款发放时购买保护性期权或结构化期权。
3. LLAMMA 式软清算 AMM：用价格 band 和套利报价完成渐进式换仓。

三条路线没有绝对优劣：

- CPPI 更通用，但路径损耗和反馈卖压更明显。
- Options 更清晰地买入凸性，但依赖期权市场深度。
- LLAMMA 更贴近 AMM 机制，但参数、band 和套利激励很复杂。

协议设计者真正要选的是：把保护成本放在 upfront premium、动态交易损耗，还是 AMM 报价折损里。

### 3. 用户界面要把 m 翻译成人话

`m` 对设计者很清楚，但对普通借款人不友好。

更好的产品表达可能是：

- 保守型：较低上行参与，较高跳空容忍度。
- 平衡型：中等上行参与，中等跳空容忍度。
- 激进型：高上行参与，较低跳空容忍度。

底层仍然用 multiplier，但界面应该显示：

- 初始风险资产比例。
- 估计 gap tolerance。
- 历史压力测试结果。
- 震荡区间下的预估 bleed。
- 触发 hard liquidation 的极端条件。

否则用户只会把它当成“不会爆仓的借贷”，最终在 whipsaw 或 gap risk 中产生错误预期。

### 4. 最适合的早期用户不是 degens

作者提到潜在目标是 treasuries、DAOs 和更保守的 allocator。

这判断是对的。

因为这类产品的卖点不是高杠杆，而是：

- 没有硬清算悬崖。
- 风险可参数化。
- collateral policy 可以打包成 vault。
- Lender 有更明确的 backstop 和 reserve。

早期如果面向高杠杆用户，很可能会被误用成“更难爆仓的杠杆工具”。更稳的路线是先把它做成保守资金能理解的抵押融资结构。

## 一句话结论

这篇文章最有价值的地方，是把链上借贷的创新焦点从“利率怎么定”转向“抵押品如何自我防御”，用 CPPI 把硬清算悬崖改造成带成本、带参数、带 backstop 的动态风险管理过程。
